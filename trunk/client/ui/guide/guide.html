User Guide

This is an example user guide window implemented as a UI component. Press I to dismiss/reopen this window.

Goals

The Goal of this UI framework is to make it easy to prototype / develop self contained UI components. In its current form, it is not a framework for use within a game client, though it may evolve in that direction over time.

Commands

  • Type /nick name to set a nickname
  • Type /youtube video-id to open a youtube video
  • Type /go url to open web page
  • Type /logout to simulate logout
  • Press I toggle this view
  • Press ENTER to start chat
  • Press / to start a slash command
  • Type /addons for a list of loaded addons

    • Draggable Components

    All windows can be moved by dragging them. Some only allow dragging from certain parts of the window. This user guide window can be dragged by grabbing hold of any window edge. The chat box can be moved by grabbing hold of the space beside the tabs, or the tabs themselves.

    Addon States

    LOADING means something went wrong, as the component never finished loading.

    LOADED means the component loaded and initialised, but did not return a public interface. This may be because it is providing functionality rather than providing an API. An example of this are the slash command handlers, they handle slash commands but don't provide a public interface

    AVAILABLE means the component loaded, initialised and returned a public interface, but did not provide a run() method and so is not considered runnable (startable).

    STARTED means the component loaded, initialised, returned a public interface which included a run() method which was called and completed successfully.
